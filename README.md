## System Overview

A Shakespeare question-answering system built on a pretrained Small Language Model (Qwen2.5-1.5B-Instruct) with Retrieval-Augmented Generation (RAG). The system is designed for users with no prior knowledge of Shakespeare and supports four interaction types:

- **Concept explanation** — "Who is Hamlet?", "What is the role of Lady Macbeth?"
- **Contextual Q&A** — "Why does Macbeth kill Duncan?", "Why does Hamlet delay taking revenge?"
- **Evidence retrieval** — every answer displays retrieved source passages with full metadata
- **Stylised generation** — short responses in Shakespearean style, clearly labelled as creative output

Corpus: _Hamlet_, _Macbeth_, _Romeo and Juliet_

```
  qwen2.5-1.5b-instruct/

  baseline_system_prompt.txt    ← baseline (prompt-only) system prompt

results/
  instructor_questions.json     ← 5 instructor-provided evaluation questions

  config.py                     ← paths and model configuration
  data_loader.py                ← loads speaker-turn JSONL files
  chunking.py                   ← speaker-turn chunking, embed_text construction
  retrieval.py                  ← EmbeddingRetriever (MiniLM + cosine similarity)
  build_index.py                ← builds and persists the retrieval index

  baseline.py                   ← baseline system (prompt-only, no retrieval)
  evaluate.py                   ← evaluation scaffold
  build_speaker_turn_chunks.py  ← generates speaker-turn JSONL from raw play JSON
  diagnose_chunks.py            ← token length diagnostics for chunking strategy
```

## Setup

### Step 1 — Install dependencies

```bash
pip install -r requirements.txt
```

### Step 2 — Build the retrieval index (first time only)

```bash
python3 src/build_index.py
```

Runs the full indexing pipeline and writes four files to `data/processed/`:

| File                | Description                                                            |
| ------------------- | ---------------------------------------------------------------------- |
| `chunks.jsonl`      | All speaker-turn chunks (unfiltered, for reference)                    |
| `embeddings.npy`    | Embedding matrix for indexed chunks                                    |
| `chunks_meta.json`  | Chunk metadata parallel to embeddings                                  |
| `scene_lookup.json` | Full scene text mapped by scene_id, used for Parent Document Retrieval |

Re-running this step during assessment is not required if the output files are already present.

> **If starting from raw data only:** Run `python3 src/build_speaker_turn_chunks.py` first to generate the speaker-turn JSONL files, then run `build_index.py`.

### Step 3 — Language model

The system uses **Qwen2.5-1.5B-Instruct**. On first run the model (~3.1 GB) downloads automatically from HuggingFace and is cached at `~/.cache/huggingface/`. An internet connection is required for the initial download only; subsequent runs load from cache.

To use a local copy instead, place the model weights at:

```
model/qwen2.5-1.5b-instruct/
```

The system detects and uses the local copy automatically if it exists.

## Running the System

### RAG chatbot — interactive mode

```bash
python3 src/rag_chatbot.py
```

Type a question at the prompt. Type `quit` or `exit` to quit. Each response displays retrieved evidence (play, act, scene, speaker, excerpt) immediately below the generated answer.

### RAG chatbot — single query

```bash
python3 src/rag_chatbot.py --query "Why does Macbeth kill Duncan?"
python3 src/rag_chatbot.py --query "Who is Hamlet?" --top_k 5
```

### Baseline system (prompt-only, no retrieval)

```bash
python3 src/baseline.py
```

### Evaluation

```bash
python3 src/evaluate.py
```

Reads from `results/instructor_questions.json` and generates a CSV evaluation template.
